# `@subsquid/ponder@0.16.6-sqd.2`

Re-cut of `ponder@0.16.6` + the SQD Portal backfill layer. Same ponder version as `0.16.6-sqd.1`;
this revision replaces the Portal layer wholesale with the current, invariant-first one.

Install: `npm i @subsquid/ponder@0.16.6-sqd.2`.

### What changed vs `0.16.6-sqd.1`

`0.16.6-sqd.1` predates the correctness wave. `-sqd.2` re-cuts the same ponder version with the
full current Portal layer — an invariant-first re-architecture plus the fix stream through the
`#47` mid-range-204 close. Grouped by theme:

- **Silent-gap / phantom-coverage elimination.** The known paths that marked a range synced
  without delivering its data now fail loud or retry: unbounded-source genesis backfill floor
  (#8, #10), head-truncated
  frontier chunks extended before serving (#5), bounded-backfill head clamp + stream-mode
  delegation fatals (#44), zero-line-200 gap (#66), and the mid-range `/finalized-stream` 204
  now treated as a transient incomplete-range that retries or fails loud (issue #47).
- **Bounded time-to-first-durable-commit (liveness).** The finalized-range fetch starts from a
  small warmup span and grows geometrically, and on a restart resumes from the persisted
  watermark, so a dense or wide-range source lands its first durable commit promptly instead of
  blocking on a full-range scan — regardless of how large the requested range is; discovery
  follows the same geometric slow-start on the watermark-following path (#50).
- **Stream-realtime correctness + reorg handling.** Strictly-increasing, parent-hash-linked
  unfinalized window; unknown-parent gap is fatal; hash-verified finality; 1-block tip orphan
  heals via HTTP 409 fork negotiation (#33); same-block child-log redelivery reconciled (#26);
  stream-mode finality floor at persisted finality; omnichain foreign-checkpoint mapping (#57);
  bounded historical→realtime cutover skip (INV-18).
- **Factory / account-source coverage.** Discovered factory children persisted in the same
  interval transaction, including on the RPC-delegation path, surviving core's transaction
  retry (#2, INV-15); write-side idempotence, no durable duplicate rows (#53); factory-range
  gate pin (#21); a receipt for every tx-filter-matched transaction (#24); the extend-path
  needed-field fatal scoped to a genuine per-filter need — no false-fatal on client-unmatched
  traces (#20) or a shared filter's receipt requirement (#94).
- **HTTP-client hardening.** Retry truncated bodies + loud dataset-start skips (#25), fail loud
  on a missing `accessList` for typed transactions (#27), zero-backoff / timeout hardening
  (#9 follow-ups, #16).
- **Invariant-first re-architecture.** `portal/` is a pure functional core behind an imperative
  shell; 18 named invariants (INV-1…INV-18, `portal/INVARIANTS.md`) asserted at runtime and
  proven by property-based + regression tests (#6, #15).

No longer the newest ponder version (`0.16.7` is), so this rev publishes under dist-tag
`ponder-0.16.6` rather than `latest`.
